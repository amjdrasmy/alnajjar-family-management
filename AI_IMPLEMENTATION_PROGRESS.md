# سجل تنفيذ تطبيق إدارة شؤون عائلة النجار

هذا الملف مخصص لاستكمال العمل لاحقًا بواسطة أي نموذج ذكاء اصطناعي أو مطوّر. يجب تحديثه بعد كل مرحلة تنفيذ أو اختبار.

## القواعد الصارمة من Speckit

- المخرج الأساسي للمرحلة الأولى هو ملف واحد: `index.html`.
- الواجهة عربية بالكامل وبتوجيه RTL.
- لا توجد أطر أو مكتبات JavaScript/CSS خارجية في المرحلة الأولى.
- التخزين عبر IndexedDB فقط، ومعزول داخل الكائن `DB`.
- اسم قاعدة البيانات: `najjar_family_db`، الإصدار `1`.
- أسماء المخازن والحقول في الكود بالإنجليزية كما في `data-model Speckit.md`.
- كل حذف أو استعادة يحتاج تأكيدًا صريحًا.
- كل قيمة مدخلة من المستخدم يجب عرضها عبر `esc()` لمنع XSS.
- بيانات الدخول التجريبية:
  - مدير: `admin / admin123`
  - عضو لجنة: `member / member123`

## الخطوة الجارية

لا توجد خطوة جارية غير مكتملة داخل هذا الدور. النسخة الحالية جاهزة للمراجعة اليدوية النهائية من المستخدم.

## خطة التنفيذ المختصرة

1. بناء HTML/CSS للتخطيط العام، شاشة الدخول، الشريط الجانبي، الشريط العلوي، المودال، التوست، ولوحة الإشعارات.
2. تنفيذ طبقة `DB` و`uid()` وتهيئة بيانات Seed.
3. تنفيذ الحالة العامة `CURRENT` و`CACHE` ومساعدات العرض.
4. تنفيذ المصادقة والصلاحيات واستعادة الجلسة.
5. تنفيذ الأقسام:
   - لوحة التحكم
   - الأفخاذ والفروع
   - الأفراد وشجرة العائلة
   - الأصناف
   - المبادرون
   - التبرعات
   - جداول التوزيع
   - المناسبات
   - الإشعارات
   - التقارير
   - سجل النشاطات
   - النسخ الاحتياطي
   - إدارة المستخدمين
6. تنفيذ البحث السريع والتصدير CSV والطباعة.
7. اختبار المسار الأساسي وتسجيل النتائج.

## ما تم تنفيذه في `index.html`

- بنية HTML5 عربية RTL مع CSS وJavaScript داخل ملف واحد.
- تصميم متجاوب مع Sidebar، Topbar، بحث سريع، مودال، Toast، وإشعارات.
- طبقة `DB` فوق IndexedDB حسب العقد:
  - `open`
  - `all`
  - `get`
  - `put`
  - `del`
  - `clear`
  - `STORES`
- بيانات Seed عند أول تشغيل:
  - مستخدما admin/member
  - أفخاذ وفروع
  - أفراد وشجرة أب/أبناء
  - أصناف ومبادرون وتبرعات
  - جدول توزيع ومناسبات وإشعار وسجل نشاط
- مصادقة محلية واستعادة جلسة من `meta.session`.
- صلاحيات Admin/Member وإخفاء الأقسام الإدارية عن العضو.
- الأقسام المنفذة:
  - لوحة التحكم
  - الأفخاذ والفروع
  - الأفراد وشجرة العائلة
  - أصناف المشاريع
  - المبادرون
  - التبرعات
  - جداول التوزيع
  - المناسبات
  - الإشعارات
  - التقارير والطباعة وCSV
  - سجل النشاطات
  - النسخ الاحتياطي والاستعادة
  - إدارة المستخدمين
- رسوم SVG داخلية:
  - خطي
  - أعمدة
  - دائري
- حماية عرض أساسية عبر `esc()` في القوالب التي تعرض مدخلات المستخدم.

## نتائج التحقق المنفذة

- [x] فحص نحوي JavaScript عبر Node المرفق مع Codex: لا أخطاء.
- [x] فحص `rg "indexedDB" index.html`: الظهور الوحيد داخل كائن `DB`.
- [x] تحميل التطبيق في المتصفح: شاشة الدخول تظهر ولا توجد أخطاء Console.
- [x] دخول المدير `admin/admin123`: يعمل وتظهر كل الأقسام.
- [x] إضافة فخذ جديد من الواجهة: تعمل ويظهر Toast نجاح.
- [x] دخول العضو `member/member123`: يعمل، ولا تظهر أزرار «إدارة المستخدمين» و«النسخ الاحتياطي» في القائمة.
- [x] المرور على أقسام العضو الأساسية: كل الشاشات تُرسم دون أخطاء Console.
- [x] المرور على أقسام المدير الإدارية: التقارير، النسخ الاحتياطي، إدارة المستخدمين تظهر دون أخطاء Console.
- [x] اختبار XSS سريع بإضافة اسم يحتوي `<script>`: يظهر كنص ولا يُنفّذ ولا يُنشئ عنصر script.
- [x] اختبار عرض 360px: زر القائمة ظاهر، لا تمدد أفقي، ولا أخطاء Console.

## ملاحظات تحقق مهمة

- أداة المتصفح الداخلية منعت فتح `file://`، لذلك تم تشغيل خادم مؤقت للاختبار على `http://127.0.0.1:8787/index.html`.
- التطبيق نفسه لا يعتمد على الخادم، ويمكن فتح `index.html` مباشرة من المتصفح حسب Speckit.
- أثناء الاختبار أضيفت بيانات داخل IndexedDB الخاصة بأصل `127.0.0.1`. هذا لا يؤثر على تشغيل الملف المباشر لأن أصل `file://` منفصل.

## ما بقي للمراجعة اليدوية

- [ ] تجربة فتح `index.html` مباشرة من المتصفح خارج أداة الاختبار.
- [ ] اختبار تصدير CSV فعليًا وفتح الملف في Excel للتحقق البصري من العربية.
- [ ] اختبار طباعة تقرير فعليًا من نافذة المتصفح.
- [ ] اختبار استعادة نسخة JSON بملف صالح وملف تالف.
- [ ] مراجعة بصرية موسعة لكل الشاشات بعد إدخال بيانات كثيرة.

## ملاحظات للمستكمل لاحقًا

- لا تستبدل IndexedDB أو تقسّم التطبيق إلى ملفات متعددة إلا إذا تم تحديث Speckit أولًا.
- استخدم `rg "indexedDB" index.html` للتأكد أن الظهور الوحيد داخل كائن `DB`.
- استخدم فتح `index.html` مباشرة في المتصفح، وليس بالضرورة عبر خادم.
- إذا كانت قاعدة IndexedDB تحتوي بيانات قديمة أثناء الاختبار، يمكن تصفيرها من وحدة التحكم:

```js
indexedDB.deleteDatabase('najjar_family_db'); location.reload();
```
