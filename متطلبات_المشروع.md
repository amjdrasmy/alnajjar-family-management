# وثيقة متطلبات النظام (SRS)
## تطبيق إدارة شؤون عائلة النجار

**الإصدار:** 1.0
**التاريخ:** تموز 2026
**النوع:** نظام إدارة داخلي لعائلة (Family Council Management System)

---

## 1. مقدمة

### 1.1 الغرض من الوثيقة
تصف هذه الوثيقة المتطلبات الوظيفية وغير الوظيفية لتطبيق إدارة شؤون **عائلة النجار**، وهو نظام يهدف إلى رقمنة أعمال لجنة العائلة: تنظيم الأفخاذ والفروع، إدارة أفراد العائلة، متابعة المبادرين والتبرعات، جدولة التوزيع، وإدارة المناسبات، مع لوحة تحكم إحصائية وتقارير.

### 1.2 نطاق النظام
النظام تطبيق ويب (يبدأ كنسخة HTML أولى) يعمل على المتصفح بتخزين محلي (IndexedDB)، ويُخطَّط لاحقًا لنسخة موبايل تعتمد **RealmDB**. لا يتطلب في مرحلته الأولى خادمًا خلفيًا؛ كل البيانات تُخزَّن محليًا على جهاز المستخدم.

### 1.3 التعريفات والمصطلحات
| المصطلح | المعنى |
|---|---|
| فخذ | التقسيم الرئيسي داخل العائلة |
| فرع | تقسيم فرعي ينتمي لفخذ |
| مبادر | شخص يقدّم تبرعًا لمشاريع العائلة |
| صنف مشروع تبرع | فئة يُجمع لها التبرع (مثل: مقبرة، ديوان، مساعدات) |
| جدول توزيع | خطة صرف/توزيع أموال أو مساعدات على مستحقين |
| SPA | تطبيق صفحة واحدة تتبدّل أقسامه دون إعادة تحميل |

## 2. المستخدمون والأدوار

### 2.1 مدير الموقع (Admin)
صلاحية كاملة: إدارة المستخدمين، كل عمليات الإضافة/التعديل/الحذف في جميع الأقسام، النسخ الاحتياطي والاستعادة، الاطلاع على سجل النشاطات.

### 2.2 عضو لجنة العائلة (Committee Member)
صلاحية تشغيلية: إضافة وتعديل البيانات (أفراد، مبادرون، تبرعات، توزيع، مناسبات)، الاطلاع على لوحة التحكم والتقارير. **بدون** إدارة المستخدمين أو حذف السجلات الحساسة أو استعادة النسخ الاحتياطية.

> ملاحظة أمنية: في نسخة HTML، التحقق من الدخول يتم على جهة العميل فقط (غير آمن للإنتاج). الأمان الحقيقي يُضاف مع الخادم الخلفي في مرحلة لاحقة.

## 3. المتطلبات الوظيفية

### 3.1 تسجيل الدخول والصلاحيات
- شاشة دخول باسم مستخدم وكلمة مرور.
- تمييز الدور (مدير / عضو لجنة) وإظهار الأقسام المسموح بها فقط.
- تسجيل الخروج، وتذكّر آخر مستخدم اختياريًا.
- حساب مدير افتراضي عند أول تشغيل: `admin / admin123`.

### 3.2 إدارة الأفخاذ والفروع
- إضافة/تعديل/حذف فخذ (اسم، رئيس الفخذ، ملاحظات).
- إضافة/تعديل/حذف فرع مرتبط بفخذ.
- عرض شجري/قائمة للأفخاذ وفروعها مع عدد الأفراد بكل منها.

### 3.3 سجل أفراد العائلة / شجرة العائلة
- بيانات الفرد: الاسم الكامل، الفخذ/الفرع، الجنس، تاريخ الميلاد، الهاتف، الحالة (حي/متوفى)، اسم الأب (للربط الشجري)، ملاحظات.
- عرض شجرة العائلة اعتمادًا على علاقة الأب–الابن.
- بحث وتصفية حسب الفخذ/الفرع/الحالة.

### 3.4 إدارة المبادرين وتبرعاتهم
- سجل المبادرين (اسم، هاتف، الفخذ، ربط اختياري بفرد).
- تسجيل تبرع (المبادر، صنف المشروع، المبلغ، التاريخ، طريقة الدفع، ملاحظات).
- عرض إجمالي تبرعات كل مبادر وسجل تبرعاته.

### 3.5 أصناف مشاريع التبرع
- إضافة/تعديل/حذف صنف (اسم، وصف، مبلغ مستهدف، لون تعريفي).
- عرض نسبة الإنجاز (المُجمّع مقابل المستهدف) لكل صنف.

### 3.6 جداول التوزيع
- إنشاء جدول توزيع (عنوان، الصنف المرتبط، التاريخ، المبلغ الإجمالي، الحالة: مخطط/جارٍ/مكتمل).
- إضافة المستفيدين وقيمة كل مستفيد.
- متابعة حالة التوزيع.

### 3.7 المناسبات والتقويم
- تسجيل مناسبة (عنوان، النوع: فرح/وفاة/اجتماع/عزاء، التاريخ، المكان، الوصف).
- عرض تقويم/قائمة للمناسبات القادمة والسابقة.

### 3.8 لوحة التحكم والإحصائيات
- بطاقات مؤشرات: عدد الأفخاذ، الفروع، الأفراد، المبادرين، إجمالي التبرعات.
- رسوم بيانية: التبرعات عبر الزمن، التبرعات حسب الصنف، توزيع الأفراد على الأفخاذ.
- أحدث النشاطات والمناسبات القادمة.

### 3.9 التقارير والطباعة والتصدير
- توليد تقارير قابلة للطباعة (تبرعات، مبادرون، أفراد، توزيع).
- تصدير البيانات إلى CSV/Excel وطباعة PDF عبر المتصفح.

### 3.10 الإشعارات
- إشعارات داخل التطبيق (جرس إشعارات + عدّاد غير المقروء).
- تنبيهات آلية عند أحداث (تبرع جديد، مناسبة قريبة، اكتمال صنف).
- *الإشعارات الفورية (Push) تتطلب خادمًا وتُضاف لاحقًا.*

### 3.11 سجل النشاطات (Audit Log)
- تسجيل من قام بأي عملية (إضافة/تعديل/حذف) ومتى وعلى أي سجل.

### 3.12 النسخ الاحتياطي والاستعادة
- تصدير كامل قاعدة البيانات كملف JSON.
- استيراد/استعادة من ملف JSON (للمدير فقط).

### 3.13 إدارة المستخدمين (للمدير)
- إضافة/تعديل/تعطيل مستخدمين وتحديد أدوارهم.

## 4. المتطلبات غير الوظيفية

- **التصميم:** واجهة عربية RTL، حديثة ومتجاوبة (Responsive) تعمل على الجوال والحاسوب.
- **الألوان الأساسية:** الأزرق (البنية والتفاعل)، الأخضر (التبرعات والإيجابي)، الأحمر (التنبيهات والمهم).
- **الأداء:** استجابة سريعة للعمليات المحلية، وتحميل أولي خفيف.
- **إمكانية الوصول:** تباين لوني مناسب، تركيز لوحة مفاتيح ظاهر، دعم حركة مخفّفة.
- **التوافق:** المتصفحات الحديثة (Chrome, Edge, Firefox, Safari).
- **الاستقلالية:** لا يعتمد على أطر خارجية في النسخة الأولى (HTML/CSS/JS خالص).

## 5. البنية التقنية

| الطبقة | المرحلة الأولى (HTML) | مرحلة الموبايل |
|---|---|---|
| الواجهة | HTML + CSS + JavaScript خالص (SPA) | React Native / Flutter |
| التخزين | IndexedDB (طبقة بيانات قابلة للاستبدال) | RealmDB |
| الرسوم البيانية | SVG مرسومة يدويًا | مكتبة رسوم أصلية |
| الإشعارات | داخل التطبيق | Push عبر خادم |

> صُمِّمت طبقة الوصول للبيانات (Data Layer) بشكل معزول بحيث يمكن استبدال IndexedDB بـ RealmDB لاحقًا دون إعادة بناء الواجهات.

## 6. نموذج البيانات (مبدئي)

- **users:** id, username, password, name, role, active
- **branches:** id, name, type(فخذ/فرع), parentId, headName, notes
- **members:** id, name, branchId, gender, birthDate, phone, status, fatherId, notes
- **donors:** id, name, phone, branchId, memberId, notes
- **categories:** id, name, description, target, color
- **donations:** id, donorId, categoryId, amount, date, method, notes
- **distributions:** id, title, categoryId, date, total, status, beneficiaries[]
- **events:** id, title, type, date, location, description
- **notifications:** id, title, body, date, read, kind
- **activity:** id, userId, action, entity, at, details

---

## 7. المراحل المستقبلية (Roadmap)

1. **م1 (الحالية):** نسخة HTML/SPA بـ IndexedDB وكل الأقسام.
2. **م2:** خادم خلفي + مصادقة آمنة + إشعارات فورية.
3. **م3:** تطبيق موبايل بـ RealmDB ومزامنة سحابية.
4. **م4:** رسائل جماعية (SMS/WhatsApp) وربط بوابات دفع.
